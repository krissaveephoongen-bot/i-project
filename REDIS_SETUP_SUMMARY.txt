                              CORE FUNCTIONS
================================================================================

Basic Caching:
  import { cacheSet, cacheGet, cacheDel } from '../lib/redis.js';
  
  await cacheSet('key', value, ttl)      // Store with TTL
  const val = await cacheGet('key')       // Retrieve
  await cacheDel('key')                   // Delete

High-Level:
  import { getOrFetch, invalidateCache } from '../lib/cache-helper.js';
  
  const data = await getOrFetch('namespace', id, fetchFn, ttl)
  await invalidateCache('namespace', id)

API Endpoints:
  GET    /api/cache/health                 // Check connection
  POST   /api/cache/set                    // Store value
  GET    /api/cache/get/:key               // Retrieve value
  DELETE /api/cache/delete/:key            // Delete key
  POST   /api/cache/flush                  // Clear all

================================================================================
                            USAGE EXAMPLE
================================================================================

In a route handler:

  import { getOrFetch, invalidateCache } from '../lib/cache-helper.js';
  
  // GET with caching
  router.get('/api/users/:id', async (req, res) => {
    const user = await getOrFetch('users', req.params.id,
      () => db.users.findById(req.params.id),
      3600  // 1 hour TTL
    );
    res.json(user);
  });
  
  // PUT with invalidation
  router.put('/api/users/:id', async (req, res) => {
    const user = await db.users.update(req.params.id, req.body);
    await invalidateCache('users', req.params.id);
    res.json(user);
  });

================================================================================
                            TROUBLESHOOTING
================================================================================

Issue: "REDIS_URL not set"
  → Add REDIS_URL to backend/.env

Issue: "Connection refused"
  → Check Redis server is running
  → Verify REDIS_URL is correct

Issue: "test-redis.mjs fails"
  → Run: node test-redis.mjs in backend/
  → Check console output for specific error

Issue: "Backend won't start"
  → Redis is optional, check logs for the error
  → Backend will start but log "⚠️  Redis will be skipped"

Issue: "Cache endpoint returns 500"
  → Check Redis connection: curl /api/cache/health
  → Restart backend service

================================================================================
                          PRODUCTION NOTES
================================================================================

Before deploying to production:

  1. Use Redis Enterprise or managed Redis service
  2. Enable persistence (AOF or RDB)
  3. Set maxmemory and eviction policy
  4. Use TLS for encrypted connections
  5. Enable Redis authentication
  6. Monitor cache hit/miss rates
  7. Set up Redis backups
  8. Implement cache warming for critical data
  9. Add monitoring and alerting
  10. Plan eviction policy (LRU recommended)
