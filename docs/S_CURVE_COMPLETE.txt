═══════════════════════════════════════════════════════════════════════════════

Step 1: Database Migration (5 minutes)
   npm run db:push

   - weight field to tasks table (numeric, default 1.00)
   - completedAt field to tasks table (timestamp)

Step 2: Assign Task Weights (Ongoing)
   - Edit each project's tasks
   - Set weight field (1-10 scale recommended)
   - Total weight represents 100% of project
   
   Example:
   - Critical Feature: 10
   - Major Feature: 5
   - Minor Task: 2
   - Bug Fix: 1

Step 3: Set Task Due Dates (Ongoing)
   - Ensure all tasks have dueDate values
   - Mark tasks as 'done' when complete
   - System auto-sets completedAt timestamp

Step 4: View S-Curve (Immediate)
   - Navigate to any project
   - Click "Overview" tab
   - See S-Curve section with:
     * 4 Status cards (Planned, Actual, Variance, Tasks)
     * Interactive line chart showing progress
