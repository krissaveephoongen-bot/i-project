================================================================================
                            QUICK START (5 MIN)
================================================================================

1. Open your server/app.js

2. Add 3 import lines:
   const RateLimiter = require('./middleware/rateLimiter');
   const requestLogger = require('./middleware/request-logger');
   const ErrorHandler = require('./middleware/error-handler');

3. Add middleware (after cors/json):
   const limiter = new RateLimiter({ maxRequests: 100 });
   app.use('/api/', limiter.middleware());
   app.use(requestLogger());

4. Add routes (before error handler):
   app.use(require('./routes/status-routes'));

5. Add error handler (MUST BE LAST!):
   app.use(ErrorHandler.middleware());

6. Restart server:
   npm run dev

7. Visit dashboard:
   http://localhost:5000/status

================================================================================
                        COMMON INTEGRATION ISSUES
================================================================================

ERROR: "404 on /status"
CAUSE: Routes not properly registered
FIX: Add statusRoutes BEFORE error handler in app.js

ERROR: "Logs directory doesn't exist"
CAUSE: logs/ folder not created
FIX: Created automatically, but run: mkdir logs

ERROR: "Rate limiting too strict"
CAUSE: Default limit is 100 per 15 minutes
FIX: Increase maxRequests in rateLimiter config

ERROR: "Health checks logging too much"
CAUSE: Checks run every 30 seconds
FIX: Increase interval in health check setup

ERROR: "Status page doesn't load"
CAUSE: Missing dependencies or import errors
FIX: Check console for import errors, restart server
