# คู่มือสำหรับผู้จัดการโครงการ (Project Manager Guide)

## ภาพรวมบทบาท

ในฐานะผู้จัดการโครงการ คุณมีสิทธิ์พิเศษในการ:
- สร้างและแก้ไขโครงการที่คุณเป็นผู้จัดการ
- จัดการงาน (Tasks) ทั้งหมดในโครงการ (สร้าง, แก้ไข, ลบ)
- ติดตามความคืบหน้าและงบประมาณ
- อนุมัติ Timesheet และค่าใช้จ่ายของทีม

**📝 หมายเหตุ:** การลบโครงการต้องมีสิทธิ์ระดับผู้ดูแลระบบ (Admin) เท่านั้น

---

## การจัดการโครงการ

### 1. สร้างโครงการใหม่

**ขั้นตอน:**
1. ไปที่หน้า **Projects**
2. คลิกปุ่ม **"New Project"** (มุมบนขวา)
3. กรอกข้อมูลโครงการ:
   - **Project Name**: ชื่อโครงการ
   - **Description**: คำอธิบายโครงการ
   - **Start Date - End Date**: ช่วงเวลาดำเนินการ
   - **Budget**: งบประมาณเป็นบาท (฿)
   - **Status**: สถานะโครงการ (Planning, Active, On Hold, Completed)
   - **Customer**: ชื่อลูกค้า/หน่วยงาน
   - **Team Members**: รายชื่อสมาชิกทีม (คั่นด้วยเครื่องหมายจุลภาค)
4. คลิก **"Save Project"**

### 2. แก้ไขโครงการ

**ขั้นตอน:**
1. ในหน้า **Projects** หาโครงการที่ต้องการแก้ไข
2. คลิกปุ่ม **"Edit Project"**
3. แก้ไขข้อมูลที่ต้องการ
4. คลิก **"Save Project"**

**📝 หมายเหตุ:** ผู้จัดการโครงการสามารถสร้างและแก้ไขโครงการได้เท่านั้น การลบโครงการต้องมีสิทธิ์ระดับผู้ดูแลระบบ (Admin)

---

## การจัดการงาน (Tasks)

### 1. เข้าสู่โหมดจัดการโครงการ

**ขั้นตอน:**
1. ไปที่หน้า **Projects**
2. คลิก **"View Details"** ที่โครงการที่ต้องการ
3. คลิกปุ่ม **"Manage Project"** (มุมบนขวา)

### 2. สร้างงานใหม่

**ขั้นตอน:**
1. ในหน้า **Manage Project** คลิกปุ่ม **"Add Task"**
2. กรอกข้อมูลงาน:
   - **Task Name**: ชื่อกงาน
   - **Description**: รายละเอียดงาน
   - **Assignee**: ผู้รับผิดชอบ
   - **Due Date**: วันที่กำหนดส่ง
   - **Weight (%)**: น้ำหนักของงานในโครงการ (0-100%)
   - **Progress (%)**: ความคืบหน้าปัจจุบัน (0-100%)
   - **Priority**: ลำดับความสำคัญ (Low, Medium, High)
   - **Status**: สถานะงาน (To Do, In Progress, Review, Completed)
3. คลิก **"Save"**

### 3. แก้ไขงาน

**ขั้นตอน:**
1. คลิกปุ่ม **"Edit"** (รูปดินสอ) ที่งานที่ต้องการแก้ไข
2. แก้ไขข้อมูล
3. คลิก **"Save"**

### 4. ลบงาน

**ขั้นตอน:**
1. คลิกปุ่ม **"Delete"** (รูปถังขยะ) ที่งานที่ต้องการลบ
2. ยืนยันการลบ

## การกำหนด Task Weight

### หนักสำคัญ
**Task Weight คือน้ำหนักของงานที่ใช้คำนวณความคืบหน้าโครงการ**

### กฎสำคัญ
- ✅ Task Weight ทั้งหมดในโครงการต้องรวมเป็น **100%** พอดี
- ❌ ถ้ารวมไม่เท่ากับ 100% ระบบจะแจ้งเตือน

### ตัวอย่างการกำหนด Weight

**โครงการ: พัฒนาเว็บไซต์**

| งาน | Weight | เหตุผล |
|-----|--------|--------|
| วิเคราะห์ความต้องการ | 10% | งานเริ่มต้น ใช้เวลาน้อย |
| ออกแบบ UI/UX | 20% | งานสำคัญ ใช้เวลามาก |
| พัฒนา Frontend | 30% | งานหลัก ใช้เวลาและทรัพยากรสูงสุด |
| พัฒนา Backend | 25% | งานหลัก ซับซ้อน |
| ทดสอบระบบ | 10% | งานตรวจสอบ |
| Deploy และฝึกอบรม | 5% | งานสุดท้าย |
| **รวม** | **100%** | ✅ ถูกต้อง |

### เคล็ดลับการกำหนด Weight
1. **พิจารณาความซับซ้อน**: งานที่ยากควรมี Weight สูง
2. **พิจารณาเวลา**: งานที่ใช้เวลานานควรมี Weight สูง
3. **พิจารณาทรัพยากร**: งานที่ใช้คนเยอะควรมี Weight สูง
4. **สอดคล้องกับแผน**: Weight ควรสะท้อนแผนงานจริง

## การติดตามความคืบหน้า

### S-Curve Analysis

**S-Curve คืออะไร?**
- กราฟแสดงความคืบหน้าโครงการเมื่อเทียบกับเวลา
- มี 2 เส้น: **Planned Progress** (แผน) และ **Actual Progress** (จริง)

**วิธีดู:**
1. ไปที่หน้า **Projects**
2. คลิก **"View Details"** ที่โครงการที่ต้องการ
3. ดูกราฟ S-Curve ในส่วน **Project Overview**

**การแปลผล:**
- **เส้นสีน้ำเงิน (Planned)**: ความคืบหน้าตามแผน
- **เส้นสีเขียว (Actual)**: ความคืบหน้าจริง

**SPI (Schedule Performance Index):**
- SPI > 1.0 = 🟢 ดำเนินการเร็วกว่าแผน
- SPI = 1.0 = 🟡 ตามแผนพอดี
- SPI < 1.0 = 🔴 ช้ากว่าแผน (ควรเร่งดำเนินการ)

### การคำนวณความคืบหน้า

**Planned Progress:**
```
PlanProgress(วันที่) = Σ(Weight ของ Task ที่ควรเสร็จภายในวันนั้น) / 100
```

**Actual Progress:**
```
ActualProgress = Σ(Task Weight × Task Progress) / 100
```

**ตัวอย่าง:**
- Task A: Weight 30%, Progress 100% → มีส่วน 30%
- Task B: Weight 40%, Progress 50% → มีส่วน 20%
- Task C: Weight 30%, Progress 0% → มีส่วน 0%
- **Actual Progress = 30% + 20% + 0% = 50%**

## การควบคุมงบประมาณ

### การติดตามค่าใช้จ่าย

**ดูสรุปงบประมาณ:**
1. ไปที่หน้า **Projects**
2. คลิก **"View Details"**
3. ดูส่วน **Budget Status**:
   - Total Budget: งบประมาณทั้งหมด
   - Total Expenses: ค่าใช้จ่ายที่ใช้ไป
   - Remaining: งบประมาณคงเหลือ

### การอนุมัติค่าใช้จ่าย

**ขั้นตอน:**
1. ไปที่หน้า **Dashboard** หรือ **Approvals** (ถ้ามี)
2. ดูรายการค่าใช้จ่ายที่รออนุมัติ
3. ตรวจสอบรายละเอียด
4. คลิก **"Approve"** หรือ **"Reject"**

**เคล็ดลับ:**
- ✅ ตรวจสอบว่าค่าใช้จ่ายสมเหตุสมผล
- ✅ ตรวจสอบว่ามีหลักฐาน/ใบเสร็จ
- ✅ ตรวจสอบว่าไม่เกินงบประมาณ
- ⚠️ ระบบจะเตือนถ้าใกล้เกินงบ

---

## การอนุมัติ Timesheet

### ขั้นตอนการอนุมัติ

1. ไปที่หน้า **Dashboard** หรือส่วน **Pending Approvals**
2. ดูรายการ Timesheet ที่รออนุมัติ
3. ตรวจสอบ:
   - ชั่วโมงที่บันทึกถูกต้องหรือไม่
   - งานที่ระบุตรงกับโครงการหรือไม่
   - ไม่เกิน 24 ชั่วโมงต่อวัน
4. คลิก **"Approve"** หรือ **"Reject"**

### เคล็ดลับ
- อนุมัติอย่างสม่ำเสมอ อย่าให้สะสม
- ตรวจสอบความเหมาะสมของเวลาที่ใช้
- สื่อสารกับทีมถ้ามีข้อสงสัย

## Best Practices สำหรับ PM

### 1. การวางแผนโครงการ
- ✅ กำหนด Timeline ที่เป็นจริง
- ✅ แบ่งงานให้ละเอียดพอเหมาะ
- ✅ กำหนด Weight ให้สะท้อนความสำคัญจริง
- ✅ มอบหมายงานชัดเจน

### 2. การติดตามโครงการ
- ✅ เช็ค S-Curve อย่างน้อยสัปดาห์ละครั้ง
- ✅ ติดตามงบประมาณสม่ำเสมอ
- ✅ อัปเดต Task Progress เมื่อมีความคืบหน้า
- ✅ สื่อสารกับทีมเป็นประจำ

### 3. การควบคุมคุณภาพ
- ✅ ตรวจสอบ Timesheet ให้ถูกต้อง
- ✅ อนุมัติค่าใช้จ่ายอย่างรอบคอบ
- ✅ ทบทวนความคืบหน้าเป็นประจำ
- ✅ แก้ไขปัญหาทันทีที่พบ

### 4. การสื่อสาร
- ✅ รายงานสถานะให้ผู้บริหารทราบ
- ✅ แจ้งทีมเมื่อมีการเปลี่ยนแปลง
- ✅ ขอความช่วยเหลือเมื่อต้องการ
- ✅ บันทึกปัญหาและแนวทางแก้ไข

## การแก้ปัญหาที่พบบ่อย

### Task Weight ไม่ครบ 100%
**ปัญหา:** ระบบแจ้งว่า Weight ไม่เท่ากับ 100%

**วิธีแก้:**
1. ตรวจสอบ Weight ของทุก Task
2. คำนวณผลรวม
3. ปรับ Weight ให้รวมเป็น 100% พอดี

### โครงการช้ากว่าแผน
**ปัญหา:** SPI < 1.0 หรือเส้น Actual ต่ำกว่า Planned

**วิธีแก้:**
1. ระบุงานที่ล่าช้า
2. หาสาเหตุ (ขาดทรัพยากร, ปัญหาเทคนิค, etc.)
3. จัดลำดับความสำคัญใหม่
4. เพิ่มทรัพยากรหรือปรับแผน
5. ติดตามอย่างใกล้ชิด

### ค่าใช้จ่ายใกล้เกินงบ
**ปัญหา:** Total Expenses ใกล้เคียงหรือเกิน Budget

**วิธีแก้:**
1. ทบทวนค่าใช้จ่ายที่จำเป็น
2. หยุดค่าใช้จ่ายที่ไม่จำเป็น
3. ขออนุมัติเพิ่มงบประมาณ
4. หาทางลดต้นทุน

---

## สรุป Key Points

✅ **จัดการโครงการ**: สร้างและแก้ไขโครงการที่เป็นผู้จัดการ
✅ **จัดการงาน**: เพิ่ม, แก้ไข, ลบ Tasks พร้อมกำหนด Weight
✅ **ติดตาม S-Curve**: วิเคราะห์ความคืบหน้าเทียบแผน
✅ **ควบคุมงบประมาณ**: ติดตามและอนุมัติค่าใช้จ่าย
✅ **อนุมัติงาน**: อนุมัติ Timesheet ของทีม

**⚠️ ข้อจำกัด**: ผู้จัดการโครงการไม่สามารถลบโครงการได้ ต้องติดต่อผู้ดูแลระบบ
