# แผนการปรับปรุงระบบให้ใช้ข้อมูลจริง (เน้น Table & CRUD)

รับทราบครับ ผมจะปรับแผนโดยตัดส่วน Kanban ออก และเน้นไปที่การจัดการข้อมูลผ่าน **ตาราง (Table View)** ที่สามารถ **สร้าง-อ่าน-แก้ไข-ลบ (CRUD)** ได้อย่างสมบูรณ์และใช้งานง่าย เพื่อให้ Admin และ Manager จัดการข้อมูลได้รวดเร็วที่สุด

## Phase 1: Task Management (แบบตาราง)
**เป้าหมาย:** เปลี่ยนหน้า Tasks จาก Kanban เป็น Table ที่จัดการงานได้ละเอียด
1.  **Task List Table:**
    *   สร้างตารางแสดงรายการงานทั้งหมด (Sortable & Filterable)
    *   คอลัมน์: ชื่องาน, โครงการ, ผู้รับผิดชอบ, วันกำหนดส่ง, สถานะ, Priority, ชั่วโมงที่ใช้
2.  **CRUD Operations:**
    *   **Create:** ปุ่ม "Add Task" เปิด Modal สร้างงาน (เลือกโครงการ, Assign คน)
    *   **Edit:** คลิกที่แถวเพื่อแก้ไขรายละเอียด (Inline Edit หรือ Modal)
    *   **Delete:** ปุ่มลบพร้อม Confirmation
3.  **Real Data:** เชื่อมต่อกับตาราง `tasks` ใน Database โดยตรง

## Phase 2: Reports & Analytics (Data-Driven)
**เป้าหมาย:** รายงานต้องคำนวณจากข้อมูลจริงใน Database เท่านั้น
1.  **Financial Report (`/reports/financial`):**
    *   ดึงข้อมูล Budget และ Actual Cost จาก Project และ Timesheets
    *   แสดงตารางสรุปกำไร/ขาดทุนรายโครงการ
2.  **Utilization Report (`/reports/utilization`):**
    *   แสดงตารางการใช้เวลาของพนักงานแต่ละคน (Billable vs Non-Billable)
    *   ดึงข้อมูลจาก `timesheets` จริง

## Phase 3: Sales & CRM (List View)
**เป้าหมาย:** จัดการรายการขายแบบ List ที่ดูง่าย
1.  **Sales Deals Table:**
    *   แสดงรายการดีลทั้งหมดในรูปแบบตาราง
    *   แก้ไขสถานะ (Stage) และมูลค่า (Amount) ได้จากหน้าตาราง

## Phase 4: System Configuration (Settings)
**เป้าหมาย:** จัดการ Master Data
1.  **Users & Clients:**
    *   ตรวจสอบหน้าจัดการ Users และ Clients ว่า CRUD ทำงานสมบูรณ์
    *   เพิ่มการจัดการ Roles และ Permissions (ถ้าจำเป็น)
